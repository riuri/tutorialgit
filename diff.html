Vendo diferenças

Para este mini-tutorial, edite alguns arquivos e repita o procedimento de commit. Assim, poderemos ver mudanças ao adicionar arquivos ao stage.

  • Modifique arquivos
  • Modo terminal
  • Modo gráfico com gitg

Modifique arquivos

Uma ideia seria comentar o Hello, World.

hello.cpp

#include <iostream>
using namespace std;

int main()
{
	// Parte que mostra a mensagem
	cout << "Olá, mundo!" << endl; // endl é fim de linha
	// O retorno 0 indica sucesso
	return 0;
}

Modo terminal

  1. Digite git add hello.cpp ou, em vez de hello.cpp, o(s) nome(s) do(s) arquivos a colocar no stage. Você já sabe que git add . adiciona todos os arquivos do diretório.

  2. Digite git diff --cached. Deve aparecer algo parecido com isso:

    $ git diff --cached
diff --git a/hello.cpp b/hello.cpp
index 39f71c6..a71cd14 100644
--- a/hello.cpp
+++ b/hello.cpp
@@ -3,6 +3,8 @@ using namespace std;
 
 int main()
 {
-       cout << "Olá, mundo!" << endl;
+       // Parte que mostra a mensagem
+       cout << "Olá, mundo!" << endl; // endl é fim de linha
+       // O retorno 0 indica sucesso
        return 0;
 }

  3. As linhas que começam com + foram adicionadas. As que começam com - foram removidas.

  4. Após conferir as mudanças, pode dar o commit como de costume.

Modo gráfico com gitg

Com o gitg aberto, após realizar as modificações, faça o seguinte:

  1. Clique na aba Commit.

  2. Arraste os arquivos de Unstaged para Staged.

  3. Clique no arquivo do stage para ver as modificações realizadas.

    Visão das modificações

    As linhas que começam com + foram adicionadas. As que começam com - foram removidas.

  4. Após conferir as mudanças, pode-se fazer a rotina de commit como de costume.
