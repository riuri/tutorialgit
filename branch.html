Criando e editando um branch

Dá para criar linhas de edição separadas, que são chamadas branches. Elas são úteis para testar ideias que poderão ou não serem adicionadas a outra linha de edição. Vejamos como criar e trocar de branch.

  • Modo terminal
  • Modo gráfico com gitg

Modo terminal

No diretório acompanhado pelo git, faça o seguinte:

  1. Digite git branch experimental. O git cria um branch chamado experimental.

  2. Para listar todos os branches locais, digite git branch.

    $ git branch
  experimental
* master

    O asterisco indica que você está agora no branch master, que é o branch inicial. O branch experimental é o que você acabou de criar.

  3. Para mudar para o branch recém criado e fazer alterações nele, digite git checkout experimental.

  4. Neste tutorial, modifique a linha do return 0; para, sei lá, return 5-5;, assim:

    hello.cpp

    #include <iostream>
using namespace std;

int main()
{
	// Parte que mostra a mensagem
	cout << "Olá, mundo!" << endl; // endl é fim de linha
	// O retorno 0 indica sucesso
	return 5-5;
}

  5. Faça o commit como de costume.

  6. Volte para o branch master com git checkout master. Observe que o arquivo hello.cpp voltou a como estava antes.

  7. Neste tutorial, mude, sei lá, a linha do cout << "Olá, mundo!" << endl; de modo que ele fique assim:

    hello.cpp

    #include <iostream>
using namespace std;

int main()
{
	// Parte que mostra a mensagem
	cout << "Olá, mundo!\n";
	// O retorno 0 indica sucesso
	return 0;
}

  8. Faça o commit como de costume.

  9. Digite git merge experimental. Isso pega as mudanças feitas no branch experimental e as junta no branch atual.

    Pode ser necessário resolver conflitos se houver mudanças nas mesmas partes. Nesse caso, digite git diff, resolva os conflitos manualmente e dê commit.

  10. Nesse ponto, pode-se remover o branch experimental digitando git branch -d experimental.

Branches são fáceis de serem criados, editados e removidos, então eles costumam ser bons jeitos de testar uma ideia nova.

Modo gráfico com gitg

Criar branches é igualmente fácil no modo gráfico. Vamos fazer o mesmo roteiro:

  1. Na aba Histórico, clique com o botão direito no último commite e escolha Criar branch.

  2. Digite o nome do branch, nesse caso, experimental.

  3. Agora você pode ver o branch que acabou de criar.

    Criando um branch local

    Esta tela mostra os branches presentes

  4. Para mudar para o branch recém criado e fazer alterações nele, clique com o botão direito do mouse em cima de experimental e clique em Checkout em cópia de trabalho.

    Pode ser necessário atualizar o gitg no menu Visualizar ▸ Atualizar ou apertando Ctrl+R.

  5. Neste tutorial, modifique a linha do return 0; para, sei lá, return 5-5;, igual ao tutorial do terminal.

  6. Faça o commit como de costume.

    Tela após o commit em experimental

  7. Volte para o branch master clicando com o botão direito em master e clicando em Checkout em cópia de trabalho. Observe que o arquivo hello.cpp voltou a como estava antes.

  8. Neste tutorial, mude, sei lá, a linha do cout << "Olá, mundo!" << endl; de modo que ele fique assim:

    hello.cpp

    #include <iostream>
using namespace std;

int main()
{
	// Parte que mostra a mensagem
	cout << "Olá, mundo!\n";
	// O retorno 0 indica sucesso
	return 0;
}

  9. Faça o commit como de costume.

    Tela após o commit em master

    A qualquer momento, você pode clicar no menu popup ao lado de Branch: e escolher Todos os branches.

    Tela de todos os branches

  10. Clique com o botão direito sobre master e clique no menu Mesclar branch com... ▸ experimental.
