Dando commit em mudanças

A cada pequena mudança relevante, é interessante dar um commit. O commit é uma espécie de backup dos arquivos no stage. O stage é uma espécie de cache das modificações que serão preservadas pelo git.

  • Crie/modifique arquivos
  • Modo terminal
  • Modo gráfico com gitg

Crie/modifique arquivos

Crie, sei lá, um hello, world.

hello.cpp

#include <iostream>
using namespace std;

int main()
{
	cout << "Olá, mundo!" << endl;
	return 0;
}

Modo terminal

  1. Digite git add hello.cpp ou, em vez de hello.cpp, o(s) nome(s) do(s) arquivos a colocar no stage.

    git add . adiciona todos os arquivos do diretório

  2. Opcionalmente, modifique mais o arquivo e coloque as mudanças no stage com git add.

  3. Digite git commit -m "Mensagem de commit". Essa mensagem deve descrever, em uma linha, as mudanças realizadas.

  4. Pronto, suas mudanças estão preservadas para a posteridade.

Repita o procedimento sempre que efetuar modificações.

Modo gráfico com gitg

Com o gitg aberto, faça o seguinte:

  1. Clique na aba Commit.

  2. Arraste os arquivos de Unstaged para Staged.

  3. Opcionalmente, modifique mais o arquivo e coloque as mudanças no stage.

  4. Digite a mensagem de commit. Essa mensagem deve descrever, em uma linha, as mudanças realizadas. Se precisar, pule duas linhas e explique mais o que as mudanças fazem.

  5. Por fim, clique no botão Commit para preservar suas mudanças para a posteridade.

Vale lembrar, repita o procedimento sempre que efetuar modificações.
